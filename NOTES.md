Block - A block is a unit of data storage which is used to store multiple transactions into one, transactions may vary from dozens to hundreds.

Hashes - Hashes are fixed size cryptographic value that are generated from the input data of block, it is generated based on data, nonce, prev. hash, timestamp, etc.

Consensus - Consensus is a mechanism in blockchain which is used to verify transactions over a blockchain network in which all the nodes in blockchain network agree on the validity of transaction and current state of blockchain.

- Q1. What is a CID and how is it generated?
        CID stands for content Id, it is a hash which is generated based on the content of the file rather than the file name. It is generated by actual contents of the file.

- Q2. Why does TraceChain store only the CID on-chain instead of the actual data?
        TraceChain stores only the CID because of cost and speed. The flow of data between blockchain costs (gas) are very higher. To mitigate this we store just the CID of the data whereas, the actual data is stored on IPFS storage.
        Accessing the actual data is very slow on chain, so this data is distributed among verious nodes which is faster to access from any node.

- Q3. What is the difference between IPFS and Pinata?
        IPFS stands for Inter Planatery File Storage, it is a protocol to store data in a decentralized form, whereas, Pinata is a platform/service provider which is based on IPFS which stores (pin) the data or file onto the IPFS.

- Q4. What happens to the data if Pinata goes down? (hint: research this — it's an interesting question)
        If Pinata goes down, the data may become unavailable.
        If the data is stored on multiple nodes then the data can be accessible from any other node.
